Conecte seu sistema de storage ASA para habilitar a comunicação em cluster, acesso à rede, conectividade de gerenciamento e conexões com os shelves de storage.



== Visão geral da cablagem de hardware

A configuração de cabos varia conforme o modelo da plataforma, pois cada modelo ASA r2 possui locais de slots, atribuições de portas e configurações de módulos de E/S exclusivos. Selecione o procedimento para o seu modelo ASA r2:

* link:cable-hardware-a1k.html["Cabo ASA A1K"]
* link:cable-hardware-a70-90.html["Cabo ASA A70 e A90"]
* link:cable-hardware-a20-30-50.html["Cabo ASA A20, A30 e A50"]
* link:cable-hardware-c30.html["Cabo ASA C30"]


Você conectará quatro tipos de conexões durante a instalação de cabos de hardware:

* *Conexões de cluster/HA*
+
Conecte os dois controladores entre si (cluster sem switch) ou aos switches de rede do cluster (cluster com switch). Essas conexões permitem a comunicação do cluster e o failover de alta disponibilidade entre os pares de controladores. O tráfego de interconexão do cluster e o tráfego de HA compartilham as mesmas portas físicas.

* *Conexões de rede do host*
+
Conecte os controladores aos hosts SAN usando Ethernet (iSCSI, NVMe/TCP) ou Fibre Channel (FC, NVMe/FC), dependendo da configuração do seu host. Essas conexões fornecem acesso aos dados para suas cargas de trabalho de storage.

* *Conexões de rede de gerenciamento*
+
Conecte os controladores à sua rede de gerenciamento para acesso de administração do sistema por meio do ONTAP System Manager e CLI.

* *Conexões para prateleiras de storage*
+
Conecte os controladores às prateleiras de armazenamento NS224 para fornecer capacidade de storage para o seu sistema.





== Recursos adicionais

Os procedimentos de cabeamento mostram configurações comuns. O cabeamento específico depende dos componentes encomendados para o seu sistema de storage. Para obter detalhes completos sobre configuração e prioridade de slots, consulte link:https://hwu.netapp.com["NetApp Hardware Universe"^].
